# Fix: Automation Agent Status Not Showing on Execution Monitor

## Debugging First

Run DB queries to determine if the issue is:
1. **No ExecutionGraphs exist** for `automation_agent` → record creation problem
2. **Graphs exist but don't show** → display problem

### Queries to run:
```sql
-- 1. Check if any automation_agent graphs exist
SELECT id, thread_id, assistant_id, title, status, started_at, completed_at 
FROM core_executiongraph 
WHERE assistant_id = 'automation_agent' 
ORDER BY started_at DESC LIMIT 10;

-- 2. Check automation_agent threads
SELECT id, name, assistant_id, is_hidden, created_at 
FROM ai_assistant_thread 
WHERE assistant_id = 'automation_agent' 
ORDER BY created_at DESC LIMIT 10;

-- 3. Check if graphs exist for automation_agent threads
SELECT g.id, g.thread_id, g.assistant_id, g.status, g.title
FROM core_executiongraph g
JOIN ai_assistant_thread t ON g.thread_id = t.id
WHERE t.assistant_id = 'automation_agent'
ORDER BY g.started_at DESC LIMIT 10;
```

## Potential Fixes (after debugging)

### If no graphs exist:
- Check Celery worker logs for errors in `auto_execute_plan` or `run_automation_agent_async`
- Check if `AIAssistant.invoke()` is being called with `thread=None` (which skips ExecutionGraph creation)
- The `_run_as_tool` method at `assistants.py:1412` calls `self.run(message, thread_id=target_thread_id)` which calls `invoke({"input": message}, thread_id=thread_id)` WITHOUT passing `thread=`. If `thread_id` is `None` (due to sub-thread creation failure), no ExecutionGraph is created.

### If graphs exist but don't show:
- Add polling to `ExecutionMonitorPage` (10s interval)
- Check if graphs have correct `assistant_id = 'automation_agent'`
