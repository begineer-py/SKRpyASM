## 第二步：理解代碼架構 (2 分鐘)

### 三個核心部分

#### 1️⃣ 掃描器 (Scanner)
**位置**：`apps/scanners/`

**做什麼**：
- Nmap：掃描 IP 和端口
- Nuclei：檢測漏洞
- Subfinder：枚舉子域名

**如何工作**：
```
創建掃描記錄 (PENDING)
    ↓
執行掃描工具 (RUNNING)
    ↓
解析結果並保存 (COMPLETED/FAILED)
```

**關鍵文件**：
- `apps/scanners/base_task.py` - 狀態管理
- `apps/scanners/nmap_scanner/` - Nmap 實現
- `apps/scanners/nuclei_scanner/` - Nuclei 實現

---

#### 2️⃣ AI 智能體 (AI Agents)
**位置**：`apps/ai_assistant/`、`apps/analyze_ai/`

**三層架構**：

```
Layer 1 (用戶交互)
  ↓ 調用工具
Layer 2 (戰略規劃) ← 讀寫 Overview 表
  ↓ 調用工具
Layer 3 (具體執行) ← 記錄 Step 和 AttackVector
```

**各層職責**：
- **Layer 1**：理解用戶命令，委派任務給 Layer 2
- **Layer 2**：制定掃描策略，調派 Layer 3 執行
- **Layer 3**：執行掃描器任務，分析掃描結果

**關鍵文件**：
- `apps/ai_assistant/assistants.py` - Layer 1 & 2
- `apps/analyze_ai/assistants.py` - Layer 3

---

#### 3️⃣ 工具 (Tools)
**位置**：各 Agent 中的 `@method_tool` 方法

**做什麼**：
將 Python 方法暴露給 AI，供 AI 調用

**示例**：
```python
class MyAgent(AIAssistant):
    @method_tool
    def execute_scan(self, target_id: int) -> str:
        """AI 會自動發現並調用這個方法"""
        return "掃描結果"
```

**關鍵文件**：
- `django_ai_assistant/langchain/tools.py` - @method_tool 定義

## 第三步：跟踪一個完整流程 (2 分鐘)

### 用戶說：「掃描 example.com」

```
1. PersonalManagerAgent 接收用戶輸入
   ↓
   
2. AI 決定：需要調用 HackerAssistantAgent
   ↓
   
3. HackerAssistantAgent 執行：
   - 查詢數據庫：list_active_targets() → 找到 example.com (ID=1)
   - 決策：調用 AutomationAgent
   ↓
   
4. AutomationAgent 執行：
   - 建立 NmapScan 記錄 (status=PENDING)
   - 調度 Celery 任務：perform_nmap_scan.delay()
   - 返回 Task ID
   ↓
   
5. Celery Worker 執行掃描：
   - 狀態轉為 RUNNING
   - 執行 nmap 命令
   - 解析結果，存入 Port 表
   - 狀態轉為 COMPLETED
   ↓
   
6. AutomationAgent 查詢結果：
   - check_nmap_results() → 返回發現的端口
   ↓
   
7. 返回給用戶：「掃描完成，發現 5 個開放端口」
```

## 第四步：重點代碼片段 (理解用)

### 掃描器如何工作

```python
# apps/scanners/nmap_scanner/tasks/__init__.py

@shared_task(bind=True)
def perform_nmap_scan(self, scan_id: int, ip_address: str):
    scan = NmapScan.objects.get(id=scan_id)
    
    # 核心：使用 Context Manager 管理狀態
    with ScannerLifecycle(scan, logger) as lc:
        # 執行掃描
        result = subprocess.run(f"nmap {ip_address}")
        
        # 存儲原始輸出
        lc.set_output(result.stdout)
        
        # 解析結果
        parse_and_save_results(scan, result.stdout)
    
    # 退出 with 時，自動設置為 COMPLETED 或 FAILED
```

---

### AI 如何調用掃描器

```python
# apps/auto/assistants/planning_agent.py

class AutomationAgent(AIAssistant):
    @method_tool
    def execute_nmap_scan(self, target_id: int, ip_address: str) -> str:
        # 建立掃描記錄
        scan = NmapScan.objects.create(
            target_id=target_id,
            ip_address=ip_address,
            status='PENDING'  # 初始狀態
        )
        
        # 調度異步任務（Celery）
        task = perform_nmap_scan.delay(scan.id, ip_address)
        
        # 返回任務 ID 給 AI（用於後續跟蹤）
        return f"掃描已啟動 (Task ID: {task.id})"
```

class HackerAssistantAgent(AIAssistant):

## 核心概念速覽

### ScannerLifecycle - 狀態管理神器

```python
# 這個 Context Manager 自動管理掃描狀態

with ScannerLifecycle(scan_record, logger):
    # 進入時：status = RUNNING, started_at = now()
    
    # ... 執行掃描 ...
    
    # 正常退出時：status = COMPLETED, completed_at = now()
    # 異常退出時：status = FAILED, error_message = exception
    # 自動保存到數據庫
```

**優點**：無需手動管理狀態，自動保證一致性

---

### @method_tool - 工具暴露裝飾器

```python
# 用 @method_tool 標記的方法自動變成 AI 工具

@method_tool
def my_function(self, param1: str) -> str:
    """工具說明（LLM 會讀這個）"""
    return "結果"

# AI 會自動：
# 1. 發現這個工具存在
# 2. 讀取文檔字符串了解用途
# 3. 根據需要調用這個方法
# 4. 將返回值作為上下文繼續推理
```

---

### 層間通信 - 共享黑板模式

```
Layer 2 ────write──→ Overview 表 ←──read──→ Layer 3
                         ↓
                    Step 表
                    AttackVector 表
```

**優勢**：不需要直接通信，通過數據庫解耦

## 常見問題速答

### Q: 掃描為什麼一直顯示 PENDING？
**A**: Celery Worker 可能沒啟動。運行 `celery -A c2_core worker -l info`

### Q: AI 為什麼沒有調用工具？
**A**: 檢查方法是否有 `@method_tool` 裝飾器，或檢查 LLM 是否識別了工具

### Q: 數據庫查詢返回為空？
**A**: 檢查表名和字段名是否正確，使用 `.values()` 可能需要指定字段

### Q: 如何添加新的掃描器？
**A**: 
1. 在 `apps/scanners/` 創建新目錄
2. 定義 `@shared_task` Celery 任務
3. 在 AI Agent 中暴露 `@method_tool`
